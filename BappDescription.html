BurpInlineDecoder is a Burp Suite extension which extracts and decodes values from HTTP responses during Intruder attacks. It writes the result into the Comment column, enabling the ability to sort, grep, and reason about responses without having to leave the Intruder UI. This extension offers an experience similar to using Grep - Extract: you choose between delimiters or a regex capture group, then select a decoder. Decoders include Base64, URL-safe Base64, Hex→Text, URL decode, Gzip/Deflate, JWT header+payload, and JSON pretty printing.

Features

  • Inline decoding into Intruder: the decoded results show up in the Comment column, adjacent to payload/length/status.
  • Two extraction modes which, like Grep-Extract, are mutually exclusive:
    • Between delimiters: Start after / End at.
    • Regex group: your regex; the first capturing group is extracted.
  • Inclusion of decoders:
    • Auto (Base64), Base64, Base64 (URL-safe)
    • Hex → Text, URL-decode
    • Gzip/Deflate (zlib & gzip heuristics)
    • JWT header + payload (pretty-printed JSON)
    • JSON pretty
  • Input sanitization for Base64/Hex, which pads and normalizes common cases.
  • Comment control: You can choose to replace the Comment or append to existing comments.
  • An optional row highlight for quick visual scanning.
  • Persistent settings across Burp sessions.

Usage

  1. Choose one extraction mode:
    • Between delimiters (start/end), or
    • Regex group (using the first capturing group).
  2. Select a Decoder:
    • "Auto (Base64)" tries standard → URL-safe Base64.
    • "Hex → Text" strips non-hex characters.
    • "Gzip/Deflate" tries gzip then zlib.
    • "JWT header+payload" pretty-prints the first two JWT parts.
    • "JSON pretty" pretty-prints valid JSON.
  3. Scope options:
    • Search headers and/or Search body.
  4. Other options:
    • Replace Comment (dont append).
    • Highlight row.
    • Comment maximum length (you can limit long values).
  5. Run Intruder → decoded values appear in the Comment column.
