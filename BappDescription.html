Inline Decoder extracts and decodes values from HTTP responses during Intruder attacks, displaying results directly in the Comment column for efficient analysis without leaving the Intruder interface. The extension provides a familiar Grep-Extract-style workflow with two mutually exclusive extraction modes and multiple decoding options.

Features

  • Live decoding during Intruder attacks with results appearing in Comment column
  • Different extraction modes (mutually exclusive like Grep-Extract)
  • Multiple decoder options
  • Configurable search scope
  • Optional row highlighting for visual identification

Usage

  1. Select one extraction method:
    • Between delimiters: specify start and end expressions
    • Regex group: provide pattern with capturing group
  2. Choose appropriate decoder:
    • Auto Base64 attempts standard then URL-safe Base64
    • Hex→Text strips non-hex characters before decoding
    • Gzip/Deflate tries gzip then zlib decompression
    • JWT decoder pretty-prints header and payload sections
  3. Configure search scope to include headers and/or response body
  4. Set comment preferences for replacing or appending results
  5. Enable extension and run Intruder attacks to see decoded values in Comment column
